## 常见问题排查

### 问题1：刷新后actions长度为0

**可能原因**：
- ❌ saveState未保存actions字段
- ❌ 点击历史记录的条件判断错误

**排查步骤**：
```javascript
// 1. 检查localStorage中是否有actions
JSON.parse(localStorage.getItem('opencode_state')).sessions[0].actions

// 2. 检查saveState函数
window.saveState.toString().includes('actions: s.actions')

// 3. 检查点击历史记录的条件
// 应该检查 actions 而不是 response
```

**解决方案**：
- 确认opencode.js第184行包含：`actions: s.actions || []`
- 确认opencode.js第539行包含：`if (s.id.startsWith('ses_') && (!s.actions || s.actions.length === 0))`

---

### 问题2：每次点击都深度加载（性能问题）

**可能原因**：
- saveState未在深度加载后调用

**排查步骤**：
```javascript
// 检查第581行是否有saveState()调用
```

**解决方案**：
- 确认opencode.js第581行包含：`saveState();`

---

### 问题3：深度加载失败

**Console错误**：
```
[History] Failed to fetch session messages: Error: Request failed
```

**可能原因**：
- 后端API未实现`/opencode/session/{id}/messages`接口
- Session ID不存在
- 网络连接问题

**解决方案**：
1. 检查后端API是否实现
2. 确认session ID格式正确（以`ses_`开头）
3. 检查网络连接

---

### 问题4：localStorage空间不足

**Console警告**：
```
[saveState] State size large: 4.80MB, approaching 5MB limit
```

**预期行为**：
- 代码会自动清理旧的空session
- 保护当前active session
- 保留5分钟内的session（宽限期）

**如果仍然失败**：
```javascript
// 手动清理localStorage
localStorage.clear()
location.reload()
```
