### 测试用例5: 防抖机制验证

**目的**: 验证saveState的100ms防抖机制

**关键代码** (opencode.js 第81-106行):
```javascript
let _saveStateDebounceTimer = null;

function saveState() {
    // 防抖：如果已经有待执行的保存，取消它
    if (_saveStateDebounceTimer) {
        clearTimeout(_saveStateDebounceTimer);
    }

    // 防抖：延迟100ms执行，避免频繁写入
    _saveStateDebounceTimer = setTimeout(() => {
        _executeSave();
    }, 100);
}
```

**验证方法**:
- 快速连续调用saveState()多次
- 预期：只有最后一次调用会在100ms后真正执行

**beforeunload事件处理** (opencode.js 第98-104行):
```javascript
window.addEventListener('beforeunload', () => {
    if (_saveStateDebounceTimer) {
        console.log('[beforeunload] Flushing pending save...');
        clearTimeout(_saveStateDebounceTimer);
        _saveStateDebounceTimer = null;
        _executeSave();  // 立即保存
    }
});
```

**测试结果**: ✅ 通过
```
[beforeunload] Flushing pending save...  ✅ 刷新前立即保存
```

## 已知限制

### 1. 旧session没有actions

**现象**: 刷新前的session（v=24之前创建）localStorage中没有保存actions

**原因**: 这些session创建时，v=24修复还未部署

**解决方案**:
- ✅ 点击历史任务时会自动深度加载
- ✅ 深度加载后会保存到localStorage
- ✅ 下次刷新就无需重新加载

**示例**:
```
[点击历史记录] actions length: 0  (旧session)
[History] Deep loading session content for: ses_xxx
[loadState] Deep load recovered messages for ses_xxx
[History] Deep load complete for: ses_xxx
```

### 2. Docker容器重启后localStorage清空

**现象**: Docker容器重启后，浏览器localStorage会被清空（不同域名或浏览器实例）

**原因**: localStorage存储在浏览器端，不在服务器

**解决方案**:
- ✅ 后端数据库仍保存完整数据
- ✅ loadState会从后端同步session列表
- ✅ 点击历史任务会深度加载恢复
