# 创建新会话422错误 - 修复报告

## 问题描述
创建新任务（会话）时报错：**422 Unprocessable Entity**

## 问题原因

### 前后端参数传递方式不匹配

**后端API期望** (app/api.py):
```python
@router.post("/session")
async def create_session(
    title: str = "New Session",
    mode: str = "auto",
    version: str = "1.0.0"
):
    # 期望从请求体（request body）获取参数
```

**前端原有调用** (static/api-client.js):
```javascript
// 错误方式：使用query参数传递
async createSession(title, mode) {
    const url = `${this.baseURL}/opencode/session?title=${title}&mode=${mode}`;
    const response = await fetch(url, { method: 'POST' });  // ❌ 422错误
}
```

### 问题分析
- 后端使用FastAPI的函数参数，默认从**请求体**读取
- 前端使用**query参数**（URL参数）传递
- FastAPI无法解析，返回422验证错误

## 修复方案

### 修改文件: `static/api-client.js`

**修改前**:
```javascript
async createSession(title = 'New Session', mode = 'auto') {
    const url = `${this.baseURL}/opencode/session?title=${encodeURIComponent(title)}&mode=${encodeURIComponent(mode)}`;
    const response = await fetch(url, { method: 'POST' });
    if (!response.ok) {
        throw new Error(`Failed to create session: ${response.status}`);
    }
    return await response.json();
}
```

**修改后**:
```javascript
async createSession(title = 'New Session', mode = 'auto') {
    const url = `${this.baseURL}/opencode/session`;
    const response = await fetch(url, {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({ title, mode })
    });
    if (!response.ok) {
        const errorText = await response.text();
        throw new Error(`Failed to create session: ${response.status} - ${errorText}`);
    }
    return await response.json();
}
```

### 关键变更
1. ✅ URL移除query参数
2. ✅ 添加 `Content-Type: application/json` header
3. ✅ 使用 `JSON.stringify()` 创建请求体
4. ✅ 改进错误处理，显示详细错误信息
