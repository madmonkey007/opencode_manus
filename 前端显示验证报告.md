## 数据流验证

### 刷新前的数据流
```
用户点击历史会话
  → 前端调用 /opencode/session/{id}/messages API
  → 后端 SessionManager.get_messages()
  → 检查内存，发现会话不存在
  → 调用 restore_session_from_db()
  → 从SQLite读取messages和steps
  → 恢复到MessageStore内存
  → 返回数据给前端 ✅
```

### 刷新后的数据流
```
用户刷新浏览器 (F5)
  → 页面重新加载
  → 内存清空
  → 用户再次点击历史会话
  → 重复上述流程 ✅
  → 数据完整恢复 ✅
```
