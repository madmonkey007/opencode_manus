# 历史记录事件显示不全问题 - 诊断与修复方案

## 问题描述

### 用户反馈
**之前分支修复了很多事件显示的问题，包括子agent展示，但实际运行中发现：**
- 一个事件都看不到
- 永远只有简单的1-2个命令
- 看不到所有事件的执行过程
- 看不到子agent的执行过程

### 预期行为
- 显示所有工具调用事件（write, edit, bash, read等）
- 显示思考过程（Think）
- 显示子agent的执行过程
- 完整的执行时间轴

## 问题诊断

### 1. 后端事件发送 ✅

**后端已正确发送SSE事件** (app/main.py):
```python
# Line 531: 时间轴更新
yield format_sse({"type": "timeline_update", "step": {...}})

# Line 534: 工具事件
yield format_sse({"type": "tool_event", "data": {"type": "tool", ...}})

# Line 552: 思考事件
yield format_sse({"type": "tool_event", "data": {"type": "thought", ...}})
```

### 2. 前端实时处理 ✅

**前端实时执行时正确处理** (static/opencode-new-api-patch.js):
```javascript
// processEvent函数处理：
- file_preview_start/delta/end → 文件预览
- phases_init/update → 阶段显示
- action/thought/error → 右侧面板显示
- timeline_event → console.log（只打印，未渲染！）
```

### 3. 历史记录加载 ❌ 问题根源

**问题**: 点击历史记录时，只加载messages，没有加载timeline

**前端可能的问题**:
1. 没有调用 `/opencode/session/{id}/timeline` API
2. 调用了但没有渲染timeline中的事件
3. 事件被filter或忽略了

## 修复方案

### 方案1: 增强processEvent处理timeline_event

**文件**: `static/opencode-new-api-patch.js`

**当前代码**:
```javascript
if (adapted.type === 'timeline_event') {
    console.log('[NewAPI] Timeline event:', adapted);
    // 可以在这里触发右侧面板更新
    return;  // ❌ 只打印，未渲染
}
```

**修复后**:
```javascript
if (adapted.type === 'timeline_event') {
    console.log('[NewAPI] Timeline event:', adapted);

    // 渲染到右侧面板
    if (window.rightPanelManager && adapted.step) {
        const step = adapted.step;
        const title = `[${step.action}] ${step.path || '工具调用'}`;
        const content = `Step ID: ${step.step_id}\nAction: ${step.action}\nPath: ${step.path || 'N/A'}\nTimestamp: ${step.timestamp}`;

        // 显示到右侧面板
        if (typeof window.rightPanelManager.show === 'function') {
            window.rightPanelManager.show();
        }
        if (typeof window.rightPanelManager.switchTab === 'function') {
            window.rightPanelManager.switchTab('preview');
        }
        if (typeof window.rightPanelManager.showFileEditor === 'function') {
            window.rightPanelManager.showFileEditor(title, content);
        }
    }
    return;
}
```

### 方案2: 历史记录加载时主动获取timeline

**文件**: 新增或修改 `static/opencode-new-api-patch.js`

**添加新函数**:
```javascript
// 加载会话的完整timeline
async function loadSessionTimeline(sessionId) {
    try {
        console.log('[History] Loading timeline for session:', sessionId);

        const response = await fetch(`/opencode/session/${sessionId}/timeline`);
        if (!response.ok) {
            console.error('[History] Failed to load timeline:', response.status);
            return;
        }

        const data = await response.json();
        const timeline = data.timeline || [];

        console.log(`[History] Loaded ${timeline.length} timeline events`);

        // 渲染每个timeline事件
        timeline.forEach(step => {
            // 模拟SSE事件格式
            const adapted = {
                type: 'timeline_event',
                step: step
            };

            // 查找对应的session对象
            const session = window.state.sessions[sessionId];
            if (session) {
                processEvent(session, adapted);
            }
        });

    } catch (error) {
        console.error('[History] Error loading timeline:', error);
    }
}
```

**在点击历史记录时调用**:
```javascript
// 在历史记录点击处理函数中添加
async function handleHistorySessionClick(sessionId) {
    // ... 现有的加载messages代码 ...

    // 新增：加载timeline
    await loadSessionTimeline(sessionId);
}
```

        if (typeof window.rightPanelManager.appendFileContent === 'function') {

        }
    }
    return;

if (adapted.type === 'subagent_end') {
